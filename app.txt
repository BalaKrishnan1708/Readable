Build a frontend-only web app called "Readable" for a dyslexia-friendly reading platform for students, teachers, and parents.

The goal:
Create a polished, playful, modern educational app UI that helps children improve reading, lets teachers upload and personalize lessons, and lets parents monitor progress. This should be a complete frontend experience with realistic mock data, strong layouts, and rich screens. Do not build backend logic. Mock all data and interactions in the frontend.

Tech and style direction:
- Use React with clean component structure.
- Use a modern design system with bold typography, rounded corners, soft shadows, playful but not childish visuals, and strong visual hierarchy.
- Use a bright education-focused palette with sky blue, indigo, emerald, amber, slate, and white.
- Use subtle animations and transitions.
- Use lucide-react style icons.
- Build responsive layouts for desktop first, but keep tablet/mobile usable.
- This is not a marketing site. The first screen after auth should be the real product.

Core product concept:
Readable is an assistive reading platform for:
1. Students
2. Teachers
3. Parents

The app helps students with reading practice, diagnostic assessments, phonics support, personalized lessons, progress tracking, and reading tools like bionic text, pacing, syllable splitting, line focus, and read-aloud.

Important:
- Frontend only.
- Use mocked API responses and local state.
- Simulate OCR extraction, Groq-based personalization, session analytics, and lesson generation with mock loading states and believable fake outputs.
- Make the app feel complete and premium, even if the data is mocked.
